Case Manager

I made this project along with four other developers for my last internship with Zayo Group.

Case Manager is an application made using Salesforce's own JavaScript Component Framework, Salesforce Lightning Components. More information about Lightning can be found here. Case Manager was made to replace the old system that developers, and a managers used to handle cases, which was simply using Salesforce. Above we can see the Developer Dashboard page, where Developers are presented with a couple of views. They can create a new case, or look at their own cases, cases they requested, or cases they need to review

We are presented with more options for the type of case when we click the "New Case Button".

Here we see an example of the required fields for making the kind of case we requested.

Above we have a case view that shows cases depending on certain criteria.

A user can click on a case to see some more information. The buttons are edit, close, send email, and reassign in that order. Clicking edit will take the user to a more detailed view.

The detailed view shows many more fields. The fields shown in the 'Basic Info' section are always shown, while Current Fields changes depending on the case status.

Selecting a search field, such as owner, or internal request contact, will bring up a search modal where the user can search through the company for the employee that's right for the field.

This is the bottom of the detailed view. Here was can change the cases status or select any of the buttons bellow. The buttons go back, save the case, insert a comment, or clone the case in that order.

Saving the case will bring up a spinner while the case is checked for errors.

Here we see other components that the user can access. Since this app was built using Salesforce Lightning Component, it works inside Salesforce One. That means that users can access the app from their desktop, tablet, or cellphone. The Case Manager makes handling cases easier while presenting a cleaner user interface.
